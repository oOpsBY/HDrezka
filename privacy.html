Политика конфиденциальности

Последнее обновление: 2026

1. Какие данные собирает расширение

Расширение HDrezka Ratings + Downloader работает локально в вашем браузере и собирает минимальные данные:

  • HTML-структура открытой страницы (анализируется локально для поиска медиаконтента и данных о фильмах/сериалах)
  • Рейтинги фильмов и сериалов (извлекаются с просматриваемой страницы, при наличии; кэшируются локально для ускорения загрузки)
  • Настройки расширения (включение/отключение функций, пользовательские предпочтения)
  • Информация о доступных видеофайлах и субтитрах, обнаруженных на странице (только при активации пользователем)
  • URL текущей вкладки (используется исключительно для логики работы расширения, например, для активации функций на определённых типах страниц)

2. Где хранятся данные

Все данные хранятся локально в вашем браузере через Chrome Storage API. Мы не используем внешние серверы для хранения, обработки или передачи пользовательских данных.

  • Настройки хранятся постоянно, пока вы их не измените или не удалите расширение.
  • Кэшированные данные (рейтинги) автоматически очищаются через 24 часа для обеспечения актуальности информации.

3. Что НЕ делает расширение

  • Не отправляет содержимое просматриваемых страниц, историю браузера или личные данные на внешние серверы.
  • Не собирает персональные данные (имя, email, пароли, данные платежных карт).
  • Не отслеживает ваши действия на сайтах, не связанных с функционалом расширения (поиск контента).
  • Не показывает рекламу и не внедряет сторонние рекламные или аналитические скрипты.
  • Не создает учетные записи пользователей.
  • Не имеет "белого списка" доменов. Анализ страницы происходит на основе её содержимого, а не адреса.

4. Разрешения расширения и их обоснование

Расширение запрашивает разрешения только для обеспечения заявленного функционала:

  • Доступ ко всем сайтам (<all_urls>) – необходимо для универсальной работы. Расширение анализирует HTML-структуру любой открытой страницы в реальном времени, чтобы обнаружить элементы кинопортала или видеоплеера и предложить соответствующий функционал (рейтинги, ссылки для скачивания).
  • Хранение данных – для локального сохранения ваших настроек и временного кэширования данных (например, рейтингов) для повышения скорости работы.
  • Скачивание файлов – для функции загрузки видео и субтитров по явному запросу пользователя (кнопка в интерфейсе расширения).
  • Активная вкладка – для определения текущего контекста (какая страница открыта) и корректной активации функций.
  • Выполнение скриптов – для чтения и анализа DOM-структуры страницы с целью извлечения данных о рейтингах и обнаружения медиа-ресурсов (видео, субтитры).

5. Принцип работы и законность функционала

Новый принцип работы: Все вычисления и анализ происходят на вашем устройстве. Расширение сканирует открытую в браузере страницу, пытаясь найти:

  • Элементы интерфейса, характерные для популярных кинопорталов (блоки с названием, описанием, постером).
  • Встроенные видеоплееры и источники видеофайлов (video теги, ссылки на .m3u8, .mp4 и другие медиафайлы).
  • Ссылки на файлы субтитров.

На основе этого анализа расширение может:

  • Отобразить агрегированные рейтинги (если они найдены на странице).
  • Предоставить кнопки для скачивания обнаруженных видео и субтитров.

Важно: Расширение является техническим инструментом, который работает только с контентом, уже доступным вам в браузере. Оно не предоставляет доступ к платному, скрытому или пиратскому контенту, который был бы вам недоступен без него. Функция скачивания доступна только для тех ресурсов, прямые ссылки на которые присутствуют в коде открытой страницы.
